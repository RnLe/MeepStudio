Dial Component Negative Number Test

This is a simple test file to verify that the Dial component works with negative numbers.

Key test cases:

  • min=-10, max=10, value=1 (should work)
  • min=-10, max=10, value=-5 (should work)
  • min=0, max=10, value=5 (should work, backwards compatible)
  • min=-1, max=1, value=0 (should work for small ranges)

The fix ensures that:

  1. When min < 0, use min directly (don't clamp to 1e-6)
  2. When min >= 0, use the original logic to avoid precision issues
  3. effectiveStep calculation uses Math.abs(internalMin) to handle negative ranges
